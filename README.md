# Agents Intensive - Capstone Project | Kaggle

Put your course learnings into action by building an agent that solves a real-world problem or improves everyday productivity.

---

## Overview

This Capstone Project is part of the [5-Day AI Agents Intensive Course with Google](https://www.kaggle.com/learn-guide/5-day-agents) (Nov 10 - 14, 2025) and is open to all participants. The project lets you apply what you’ve learned by building AI agents in one of four tracks:
- Concierge Agents
- Agents for Good
- Enterprise Agents
- Freestyle

**Submissions are due by Dec 1, 2025, 11:59 AM PT.**  
Top projects will win Kaggle swag, social media recognition, and all participants will receive a badge and certificate.

- **Start:** 9 days ago
- **Close:** 8 days to go

---

## Description

This capstone project is an opportunity to apply your course knowledge:
1. **Select a [track](https://www.kaggle.com/competitions/agents-intensive-capstone-project#tracks-and-awards)**
   - Example: Concierge Agents
2. **Formulate a problem and solution pitch**
   - Example: Automated Blog Writer Agent
3. **Develop and publish your agent code publicly**
   - Example: GitHub or Kaggle Notebooks
4. **Prepare a writeup articulating your agent’s value**
   - Example: “Reduced blog writing process by 10 hours/week”
5. **Bonus:** Publish a related video
6. **Submit your writeup by Dec 1, 2025, 11:59AM PT**
   - See ["How Do I Make A Submission"](https://www.kaggle.com/competitions/agents-intensive-capstone-project#how-do-i-make-a-submission)

## Features to Include in Your Agent Submission

Apply at least three (3) of the following concepts:
- Multi-agent systems (LLM agents, parallel/sequential/loop agents)
- Tools (MCP, custom tools, built-in/OpenAPI tools, long-running operations)
- Sessions & Memory (state management, long-term memory)
- Context engineering (e.g. compaction)
- Observability (logging, tracing, metrics)
- Agent evaluation
- A2A Protocol
- Agent deployment

## Evaluation

Submissions evaluated in three categories (max 100 points):
- **Category 1: The Pitch** (max 30 pts)
  - Core Concept & Value: 15
  - Writeup clarity: 15
- **Category 2: Implementation** (max 70 pts)
  - Technical implementation: 50
  - Documentation: 20
- **Bonus** (max 20 pts)
  - Effective Use of Gemini: 5
  - Agent deployment evidence: 5
  - YouTube video: 10

#### Example Criteria:
| Criteria (points)          | Description |
|----------------------------|-------------|
| **Pitch:** (30)            | Why & what of your project |
| **Core Concept & Value:** (15) | Central idea, relevance, agents use |
| **Writeup:** (15)          | Articulation of problem, solution, architecture |
| **Implementation:** (70)   | Quality, technical design, AI integration |
| **Tech Implementation:** (50) | At least 3 key course concepts, code quality, comments |
| **Documentation:** (20)    | Readme, problem/solution, diagrams/instructions |
| **Bonus:** (20)            | Gemini use, deployment, video |

## F.A.Q.

- **Can I make multiple submissions?**  
  No, only ONE submission allowed.
- **Can I use other languages with ADK?**  
  Yes: Python, Java, Go.
- **Is cloud deployment required?**  
  No, but you get bonus points if you do.
- **Can I submit via Colab/Kaggle Notebooks?**  
  Yes, GitHub repo or Kaggle/Colab notebook is accepted.
- **How to join/add people to teams?**  
  Add teammates via Team tab; join via Discord.
- **Can I submit if my code isn't complete?**  
  Yes, all criteria still evaluated.
- **Is English required for submissions?**  
  Yes, for judging fairness.
